可变参数宏，最高支持64个参数，这个已经足够用了。 

宏展开的流程
第一步：首先用实参代替形参，将实参代入宏文本中
第二步：如果实参也是宏，则展开实参
第三步：最后继续处理宏替换后的宏文本，如果仍包含宏，则继续展开
注意：如果在第二步，实参代入宏文本后，实参之前或之后遇到#或##，实参不再展开

这个宏定义的展开是这样的。
以FL_ARG_COUNT(1,2,3) 为例子， 先将##__VA_ARGS__替换成FL_INTERNAL_ARG_COUNT_PRIVATE(0, 1, 2, 3, 64, 63,...., 2, 1, 0) 先用实参代替了形参。这里参数个数就是65+3个，
然后由于FL_INTERNAL_ARG_COUNT_PRIVATE 是一个可变参数宏，前面65个是固定参数，上面的0就会代替_0, 1代替_1，2代替_2，3代替_3，由于超过了65个参数了，所以上面的FL_INTERNAL_ARG_COUNT_PRIVATE传递的最后几个参数就变成可变参数了。
当调用FL_ARG_COUNT(1,2,3)，那么FL_INTERNAL_ARG_COUNT_PRIVATE就有3个可变参数，那么最后的展开就成了FL_INTERNAL_ARG_COUNT_PRIVATE(.., 3, 2, 1, 0)，其中2,1,0是可变参数。那么最后这个宏展开得到的N就是3，刚好就是参数个数
